## 🛠 How to Build Your Own Portfolio (Step-by-Step)

If you'd like to create a portfolio similar to this one, follow these steps:

### 1. Preparation
- **Project Structure**: Create a folder with an `index.html`, `style.css`, and `script.js` file.
- **Assets**: Place your profile photo and any images in an `assets/` subfolder.

### 2. Craft Your Content (`index.html`)
- Use **semantic HTML5** (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`) to help with SEO and screen readers.
- Focus on describing **impact** rather than just listing features. Use terms like "optimization," "scalability," and "strategy."

### 3. Design Your System (`style.css`)
- Define **CSS Variables** (`:root`) for colors, spacing, and typography to maintain consistency.
- Use **Flexbox** and **Grid** for advanced layouts that work across all screen sizes.
- Add a **Hero section** with a unique visual element (like a rotating gradient ring) to grab immediate attention.

### 4. Add Interactivity (`script.js`)
- Implement a **mobile menu toggle** with accessible ARIA states.
- Ensure **focus trapping** (using `Tab` and `Shift+Tab`) so keyboard users don't get stuck in the navigation menu when it's open.
- Add **Smooth scrolling** for intra-page links to provide a premium feel.

## 🌐 Deployment to GitHub Pages

To make your portfolio live and shareable, follow these instructions to deploy it on GitHub:

### 1. Create a GitHub Repository
- Log in to your [GitHub Account](https://github.com/).
- Click **New** to create a repository. Name it something like `my-portfolio`.
- **Note**: Ensure the repository is **Public**.

### 2. Upload Your Code
- **Via Command Line (Recommended)**:
  ```bash
  git init
  git add .
  git commit -m "Initial commit: Deploying my portfolio"
  git branch -M main
  git remote add origin https://github.com/your-username/my-portfolio.git
  git push -u origin main
  ```
- **Via GitHub Web UI**: You can also drag and drop your `index.html`, `style.css`, `script.js`, and `assets` folder directly into the repository page.

### 3. Enable GitHub Pages
- Go to the **Settings** tab of your GitHub repository.
- On the left sidebar, click **Pages**.
- Under "Build and deployment" > "Branch," select **main** (or the branch you uploaded to).
- Ensure the folder is set to **/(root)**.
- Click **Save**.

### 4. Visit Your Site!
- GitHub will now start building your site. Within 1–2 minutes, a header will appear saying: **"Your site is live at https://your-username.github.io/my-portfolio/"**.
